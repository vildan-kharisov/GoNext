# TEST_STAGE8.md

## Этап 8 — проверка MVP

Цель: пройти ключевой пользовательский сценарий и зафиксировать результат тестирования.

## Предусловия

- Приложение запускается (`npm run start -- --clear`).
- В настройках доступна кнопка: `Сбросить локальные данные (MVP тест)`.

## Чек-лист сценария

1. Создать места:
   - открыть `Места`;
   - добавить 2-3 места;
   - проверить сохранение карточки и фото;
   - при необходимости открыть место на карте.
2. Создать поездку:
   - открыть `Поездки`;
   - создать поездку с названием и датами;
   - опционально пометить как текущую.
3. Сформировать маршрут:
   - добавить места из базы;
   - добавить новое место прямо из деталей поездки;
   - изменить порядок (`вверх/вниз`).
4. Вести дневник поездки:
   - отметить пункт как посещенный;
   - заполнить дату визита и заметки;
   - прикрепить фото для пункта маршрута.
5. Проверить режим `Следующее место`:
   - при текущей поездке показывается первое непосещенное место;
   - работают кнопки открытия в карте и навигаторе;
   - после отметки всех пунктов показывается состояние "Маршрут завершен".

## Проверка устойчивости к перезапуску

1. Перезапустить приложение.
2. Убедиться, что данные (места, поездки, маршрут, статусы посещения, фото) сохранились.
3. Для повторного прогона использовать сброс данных в `Настройки`.

## Технические проверки (выполнено)

- `npx tsc --noEmit` — без ошибок.
- `npx expo export -p web` — сборка успешна.

## Ручная проверка платформ

- Android: требуется ручной прогон на устройстве.
- iOS: требуется ручной прогон на устройстве/симуляторе.
