# PLAN.md

## Поэтапный порядок реализации GoNext (MVP)

### Этап 1. Базовая настройка проекта
- Проверить структуру Expo Router + TypeScript проекта.
- Подключить и проверить React Native Paper как основу UI.
- Настроить базовую навигацию и корневой layout приложения.
- Сформировать стартовый экран с кнопками: `Места`, `Поездки`, `Следующее место`, `Настройки`.

### Этап 2. Проектирование данных и локального хранилища
- Определить модели данных `Place`, `Trip`, `TripPlace` по полям из `PROJECT.md`.
- Спроектировать SQLite-схему: таблицы, связи, индексы, ограничения.
- Реализовать слой доступа к данным (репозитории/сервисы) для CRUD-операций.
- Добавить инициализацию БД и миграции при запуске приложения.

### Этап 3. Режим «Места»
- Создать экран списка мест с отображением основных данных.
- Реализовать создание и редактирование места (название, описание, координаты, флаги). Координаты обычно задаются в
формате в формате десятичных градусов (Decimal Degrees,DD)
- Добавить просмотр карточки места и открытие координат на карте.
- Реализовать привязку локальных фотографий к месту.

### Этап 4. Режим «Поездки»
- Реализовать список поездок и создание новой поездки.
- Добавить экран деталей поездки: даты, описание, упорядоченный маршрут.
- Реализовать добавление мест в поездку (из базы и создание нового места при необходимости).
- Добавить изменение порядка мест в маршруте и отметку `visited`.
- Реализовать заметки, дату посещения и фото в контексте `TripPlace`.

### Этап 5. Режим «Следующее место»
- Реализовать выбор активной поездки (`current = true`).
- Добавить алгоритм определения следующего места: первое `visited = false` по `order`.
- Создать экран «Следующее место» с краткой информацией.
- Реализовать кнопки открытия в карте и навигаторе.

### Этап 6. Офлайн и файловая система
- Проверить работу всех ключевых сценариев без интернета.
- Организовать хранение фотографий в файловой системе приложения.
- Добавить обработку ошибок чтения/записи данных и отсутствия геоданных.

### Этап 7. UI/UX и удобство использования
- Привести интерфейсы к единому стилю React Native Paper.
- Добавить состояния загрузки, пустых списков и ошибок.
- Упростить пользовательские потоки без сложных вложенных меню.

### Этап 8. Тестирование MVP
- Протестировать полный сценарий:
  1) создать места;
  2) создать поездку;
  3) добавить места в маршрут;
  4) отмечать посещения;
  5) проверить экран «Следующее место».
- Провести ручные проверки на Android и iOS.
- Проверить устойчивость к перезапуску приложения (данные сохраняются локально).

### Этап 9. Подготовка к релизу MVP
- Провести финальную чистку кода и проверку типизации.
- Обновить документацию по запуску и структуре проекта.
- Подготовить сборку MVP для демонстрации.
